# 里氏替换原则


## 里氏替换原则：

父类能出现的地方，可以用子类代替。

补充一下：子类拥有父类的所有属性和方法。

**继承的优点：**
*提高代码的重用性。
*提高代码的可扩展性。
*提高产品的开放性。
**缺点：**
*侵入性太强。只要继承就要拥有父类的方法和属性
*耦合性高，父类的修改需要考虑子类


java 采用单继承，C++采用多继承。但java 有接口来补充多继承。因为类可以实现多个接口。

在写代码的时候，我们要用抽象的思维，我们使用抽象类，接口来处理，至于调用什么样实现类。由调用者决定。

user.call(new IPhone)  或者 user.call(new xiaomi())      #user.call(Phone phone)

用户使用苹果手机或者小米手机都是使用手机

**java 的三大特征： 封装、继承、抽象。**

在子类不能完成父类的方法的时候，就要断开父子继承关系，采用依赖、聚集、组合来代替继承。

**采用里氏替换原则的目的：**
*1、增强程序的稳健性
*2、版本可以很好的保持兼容性
*3、使用父类作为参数，传递不同子类完成业务逻辑，很不错
